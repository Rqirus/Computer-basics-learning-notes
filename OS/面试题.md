#####进程和线程的区别

```
进程和线程的主要区别在于它们是不同的操作系统资源管理方式。
{
        线程的粒度比进程还小;
        进程是资源拥有的基本单位
        {
                进程有独立的地址空间，一个进程崩溃后，在保护模式下不会对其他进程产生影响，
                而线程只是一个进程中的不同执行路径;
         },
        线程是独立调度与独立运行的基本单位;
        由于线程基本不拥有系统资源，所以在进行切换时，线程切换的开销远远小于进程
        {
                线程有自己的堆栈局部变量，但线程没有单独的地址空间，
                一个线程死掉就等于整个进程死掉
                对于一些要求同时进行并且要共享某些变量的并发操作只能用线程，不能用进程。
        }
    
}
```
